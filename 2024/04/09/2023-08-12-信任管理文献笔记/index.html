A Subjective-trust based Trust Management Model in Ad Hoc Networks

一、研究背景

文章首先介绍了Ad Hoc网络的概念、特征及应用场景。Ad Hoc网络是一种由移动节点组成的无线通信网络,它可以自主组网,不依赖固定的基础设施。由于节点都是移动的,网络拓扑结构是动态变化的。作者指出Ad Hoc网络广泛应用于军事、救灾等领域。

然后文章分析了Ad Hoc网络面临的安全挑战。由于缺少中心管理和控制,每个节点既是主机又是路由器,网络容易受到攻击。传统网络的许多安全机制也不适合Ad Hoc网络的特点。因此如何建立安全可靠的Ad Hoc网络信任管理机制就成为一个关键问题。

文章回顾了已有的信任管理相关研究。提到信任管理通常包含安全策略制定、证书获取和证书调整三个方面。但现有研究存在依赖证书中心等问题,不适合Ad Hoc网络。所以文章提出要设计一种适合Ad Hoc网络特点的信任管理模型。

二、系统设计

文章在TCP/IP协议栈的基础上设计了Ad Hoc网络体系结构,将信任管理放在应用层,使用UDP传输。

然后给出了包含四个模块的信任管理逻辑模型,分别是信任度评估、数据收集、数据存储和行为指导模块。按照功能划分了包含多个组件的信任管理功能模型。

信任评估是核心功能,文章设计了一种整合直接信任和推荐信任的评估算法。直接信任基于节点之间的历史交互计算,推荐信任则综合周围节点的推荐。

三、实现机制

在系统实现方面,文章使用了服务原语来表示不同层次之间的交互过程。

主观信任评估是模型的核心函数,文章详细阐述了其计算方法。首先根据历史交互行为判断波动点,然后根据波动类型选择合适的预测模型计算主观信任。

在推荐信任计算中,文章引入了反馈机制来评估推荐者的可信程度,通过验证推荐信息的准确性来抵御恶意推荐攻击。

四、效果验证

文章通过分析验证了该模型的以下效果:

模型可以分布式运行,每个节点可以自主管理信任,不依赖中心节点。

不依赖传统的认证中心,适合Ad Hoc网络的去中心化特点。

算法整合了直接、推荐信任的计算,可以更准确评估节点信任度。

通过反馈机制抵御恶意推荐攻击,提高了安全性。

模型为各个Ad Hoc网络节点提供了信任管理框架和方法。

综上所述,文章在研究Ad Hoc网络背景下,设计了一个分布式的、适合该环境的主观信任模型和信任管理框架,旨在提高Ad Hoc网络的安全性和可靠性。从背景到模型设计再到实现机制,文章进行了系统详细的描述和论证。

An Attack-Resistant Trust Inference Model for Securing Routing in Vehicular Ad Hoc Networks

一、研究背景

  1. 介绍车载自组网的发展背景和应用需求。车载网络可以实现车辆间的信息交流,但节点高速移动带来的路由安全是需要解决的难题。

  2. 分析现有的基于加密和PKI的安全方案存在的问题,如依赖中心化服务器,网络开销大,内部攻击难防等。

  3. 指出基于信任的安全机制可以有效complement现有方案的不足,已经成为一个热点方向。

  4. 回顾了已有的车载网络信任管理研究,包括信任计算模型和基于信任的路由协议。

  5. 指出现有信任模型抗攻击能力较弱,路由协议也不够成熟,还需进一步的研究。

二、系统设计

  1. 构建了包含主观信任和推荐信任的车载网络信任推理模型。

  2. 主观信任基于节点之间的历史交互行为计算。推荐信任综合周围节点的推荐。

  3. 通过整合两者计算节点的综合信任度,用于评估节点是否可信。

  4. 目标是构建一个准确、高效的信任计算框架,快速识别出恶意节点。

三、实现机制

  1. 在主观信任计算中,使用一种基于波动识别的SCGM加权马尔可夫预测算法。

  2. 该算法引入了波动趋势的概念,可以识别时移攻击等复杂攻击行为。

  3. 在推荐信任计算中,提出一种基于反馈的推荐者可信度评估方法。

  4. 借助反馈机制验证推荐的准确性,可以有效抵御恶意推荐攻击。

四、效果验证

  1. 使用NetLogo平台模拟验证了模型的攻击识别能力。结果显示模型可以准确找出攻击节点。

  2. 使用SUMO和NS-2平台,对比评估了不同的信任感知路由协议。

  3. 结果表明基于该信任模型的LTMRP协议性能最好,可以建立安全可靠的通信路径。

五、总结

  1. 文章在研究车载网络背景下,设计了一个抗攻击的节点信任评估模型和基于该模型的安全路由协议。

  2. 模型通过预测和反馈机制增强了安全性。文章从背景到模型再到验证进行了系统的阐述。

  3. 该信任计算框架可以快速识别恶意节点,为车载网络的安全防护提供了一个有效参考。

  4. 相比现有研究,该模型具有更强的抗攻击能力,也为车载网络信任管理提供了一个实用的解决方案模板。

An Empirical Analysis to Assess the Operational Design Domain of Lane Keeping System Equipped Vehicles Combining Objective and Subjective Risk Measures

一、研究背景

文章首先介绍了车辆自动化的分类,车道保持系统属于SAE 2级半自动驾驶。每个自动化级别都有其操作设计域(ODD)。不同OEM的ODD定义可能有差异,这可能导致驾驶员产生困惑。文章指出驾驶员对系统能力的理解与OEM定义之间的差异,可能影响驾驶员的信任和风险感知。

文章回顾了现有关于车道保持系统对驾驶员行为影响的研究较少,主要集中在预警型系统上。且多数研究基于模拟数据。因此需要一种综合目标和主观风险度量的真实道路测试方法来评估ODD。

二、实验设计

选择包含ODD范围内和范围外的测试路段,进行仪器化改装。

问卷调查获得驾驶员的主观态度。

共19名驾驶员参与,测试车型为特斯拉Model S。

使用LIDAR、摄像头等传感器采集车辆行驶数据。

三、数据分析

分析不同ODD情况下车辆相对车道中心的平均偏差和方差。

应用概率风险场方法计算目标风险。

调查分析驾驶员的信任感和ODD认知。

综合上述评估ODD的符合性。

四、结果讨论

发现驾驶员对ODD的认知与OEM定义存在误配,可能影响交互安全。

提出OEM改进对驾驶员的告知,减少误配发生。

该评估方法可帮助OEM更明确地定义LKAS的ODD。

研究存在样本量较小等限制。后续研究可扩大样本并考虑动态风险因素。

综上所述,文章提出一种新的方法评估车道保持系统的ODD,并通过真实道路测试获得了有价值的研究发现,具有重要的学术参考价值。

Filtering Malicious Messages by Trust-Aware Cognitive Routing in Vehicular Ad Hoc Networks

一、研究背景

  1. 介绍车载自组网的应用背景,它可以实现车辆之间的信息交换,提高道路安全性。

  2. 分析由于网络开放性,内部攻击者可以传播错误信息,导致安全问题。

  3. 指出车载网络的高动态特性使现有安全解决方案效果不佳。

  4. 回顾现有车载网络信任管理研究,多是基于密码技术,不适合内部攻击。

  5. 提出需设计一种可对信息进行可信度评估的认知信任管理机制。

二、系统设计

  1. 设计包含一致性验证和行为验证两个模块的认知信任管理框架。

  2. 一致性验证利用信息冗余进行过滤,建立主观逻辑模型评估信息可信度。

  3. 行为验证评估事件报告节点的可信度。

  4. 额外设计了一个离线的恶意节点检测模块。

三、实现机制

  1. 一致性验证通过CAM和DENM获得冗余信息,避免单一来源的攻击影响。

  2. 对转发报文,计算路径可靠性;对源报文,考虑报告节点行为。

  3. 恶意节点检测基于主观逻辑和加权累积证据模型,可以有效抵御误报。

四、效果验证

  1. 仿真结果显示,该方案准确率明显优于基准模型。

  2. 在不同攻击场景和节点密度下,准确率也很高。

  3. 恶意节点检测的各项指标如精确率、召回率和F1值也很高。

  4. 结果证明该方案可以有效识别恶意信息并检测恶意节点。

五、总结

  1. 文章设计了一种车载网络的认知信任管理和恶意节点检测机制。

  2. 通过冗余信息利用和主观逻辑技术实现了安全可靠的信息路由选择。

  3. 相比现有研究,该方案可以抵御内部攻击,适用于车载网络环境。

  4. 综上所述,文章对车载网络安全问题进行了有效的技术研究,设计了一种可靠的信息过滤和恶意节点检测方案。

An Intelligent Transaction Migration Scheme for RAFT-Based Private Blockchain in Internet of Things Applications

一、研究背景

文章首先介绍了区块链技术可以为物联网应用提供安全可靠的数据存储。但物联网场景的数据量巨大,如何降低写入区块链的延迟仍然是一个挑战。文章指出通过MEC和Raft共识,可以在基站部署区块链网络。但仍需进一步研究如何减少写入延迟。

文章回顾了现有利用MEC技术进行负载分担的相关工作,但多基于PoW共识。Raft算法更适合物联网场景,但相关优化研究较少。文章将在Raft区块链中研究交易迁移问题。

二、系统设计

考虑基于MEC的Raft区块链网络,节点分为多个集群,每个集群内选举一个Leader负责收集交易和生成区块。

定义不同集群之间可以进行交易迁移,目标是最小化所有集群中的最大延迟。

将问题建模为马尔可夫决策过程,采用深度确定性策略梯度方法获得交易迁移的最优策略。

三、实现机制

设计包括状态、动作、转移函数和奖励函数的MDP模型。

使用Actor-Critic网络结构及目标网络近似Q值函数和策略函数。

为处理环境动态变化,使用动作优化选择最优的离散动作执行。

详细描述了训练过程及参数设置。

四、效果验证

仿真结果显示,该方案可以快速收敛,显著减少不同负载情况下的延迟。

比较结果表明,相比固定迁移方案,该方案可以根据环境调整迁移策略,最大降低58%的延迟。

当发生异常导致负载增加时,可以快速将交易迁移到其他闲置的集群中。

综上所述,文章在Raft区块链物联网场景下,设计了一种智能的交易迁移机制,可以根据环境动态调整,显著减少写入延迟。

Weighted RAFT: An Improved Blockchain Consensus Mechanism for Internet of Things Application

一、研究背景

文章首先介绍了物联网的发展背景,区块链技术可以很好地解决数据隐私和安全问题。Raft算法简单高效,适合物联网设备使用。但原始Raft算法节点选举概率相同,没有考虑节点性能差异。所以需要设计一个结合节点性能的Raft算法。

文章回顾了相关改进Raft算法的研究,如优化选举过程等。但现有研究没有考虑节点的通信和计算状态来优化Raft。本文拟在物联网场景中,设计一种权重Raft算法。

二、系统设计

  1. 系统包含物联网终端、网关节点和服务器。网关节点组成区块链网络。

  2. 在心跳包中收集各节点转发数据量和信道质量,计算节点权重。

  3. 权重越大的节点,在选举阶段得到的超时时间越短,成为Leader的概率越大。

三、实现机制

  1. 心跳包机制:Leader周期发送心跳包,Follower收集自身状态信息返回。

  2. 权重计算:考虑数据转发量和信道质量,合理设置权重计算公式。

  3. 选举阶段:根据权重值确定各节点的超时时间,优先选举权重较大的节点。

四、效果验证

  1. 仿真结果显示,与原始Raft相比,该机制可以减少24%的总延迟。

  2. 当异常发生时,可以快速将交易迁移到闲置的集群中。

  3. 权重机制保持了可用性,也显著提升了系统效率。

综上所述,文章设计了一种权重化的Raft共识算法,可以根据节点性能动态选择Leader,改善了物联网场景下的效率。

Trust Assessment in Vehicular Social Network Based on Three-Valued Subjective Logic

一、研究背景

  1. 介绍车联网的发展背景,车辆可以交换数据来扩大感知范围。

  2. 指出交换的数据可信度无法保证,需要评估信息发布车辆的可信度。

  3. 回顾可信度通常建模为一个实数或统计分布,但在复杂网络中效果不佳。

  4. 提出构建车辆社交网络,应用社交网络中的信任传播和融合方法。

二、系统设计

  1. 使用三值主观逻辑表示车辆之间的信任关系。

  2. 直接相互作用的车辆,根据信息一致性计算主观信任。

  3. 无直接作用的车辆,利用推荐计算间接信任。

三、实现机制

  1. OpinionWalk算法在车辆社交网络中搜索信任传播路径。

  2. 将矩阵乘法中的加法和乘法运算替换为信任的融合和衰减运算。

  3. 动态网络下,使用社区检测和标签传播算法将车辆划分到不同社区。

  4. 社区内部使用OpinionWalk算法计算车辆间的信任。

  5. 跨社区车辆通过汇聚各自的社区信息计算信任。

四、效果验证

  1. 仿真验证了直接和间接信任计算的准确性。

  2. 社区划分仅增加约5%的误差,但大幅减少了计算时间。

  3. 结果表明该方案可以高效准确地评估车辆网络中的信息可信度。

五、创新点

  1. 在车辆网络中首次引入社交网络视角进行可信度评估。

  2. 设计了基于三值主观逻辑的信任计算框架。

  3. 提出OpinionWalk算法进行大规模信任传播计算。

  4. 设计了动态社区划分来支持车辆网络的可信度评估。

综上,文章在车辆网络场景下,设计了一套完整的可信度评估机制,从背景到模型设计到实现和验证都进行了较为完整的阐述。

Trust Assessment in Vehicular Social Network Based on Three-Valued Subjective Logic

一、研究背景

  1. 文章指出车辆网络信息的可信度对确保驾驶安全至关重要,需要评估车辆及信息的可信度。

  2. 传统安全性方法无法评估信息发布者(车辆)的可信度。

  3. 文章将车辆信息交换视为一种社交行为,构建车辆社交网络,在网络内部进行可信度评估。

二、系统模型

  1. 使用三值主观逻辑模型表示车辆之间的信任关系。

  2. 直接交互的车辆,根据信息一致性计算主观信任。

  3. 无直接交互的车辆,利用信任传播和融合机制计算推荐信任。

三、信任评估步骤

  1. OpinionWalk算法在车辆社交网络中搜索可能的信任传播路径。

  2. 替换矩阵运算中的加法和乘法为信任融合和衰减运算。

  3. 动态网络下,将车辆划分到社区内部进行高效的可信度评估。

四、效果验证

  1. 仿真结果表明,该方案可以有效评估车辆的主观和客观可信度。

  2. 社区划分后,可信度评估错误率仅增加约5%,但时间复杂度下降显著。

  3. 结果证明该方案可以高效准确地评估车辆社交网络中的可信度。

综上,文章在车辆社交网络背景下,设计了一种基于三值主观逻辑的分布式可信度评估机制,使车辆能够评估信息的可信性。

Trust-aware Control for Intelligent Transportation Systems

一、研究背景

  1. 文章指出智能交通系统是一个多智能体系统,车辆和交通基础设施都可以看作智能体。

  2. AI方法可以协调不同智能体,提高系统的安全性和效率。但是也使系统容易受到攻击。

  3. 需要一种方法来量化智能体的可信度,以实现可信任的协调和控制。

二、系统模型

  1. 系统包含中心交通控制器和多个车辆智能体。

  2. 控制器根据车辆的请求批准或拒绝它们进入交叉路口。

  3. 车辆应严格遵守控制器指令,否则会威胁系统安全。

三、信任评估步骤

  1. 信任度计算模块根据车辆行为收集证据,使用主观逻辑计算信任度。

  2. 信任度聚合模块将多个本地信任度评估结果进行融合。

  3. 证据来自道路感知单元的直接观察,以及其他车辆的间接推荐。

四、实验结果

  1. 与传统方法相比,加入信任度后可以大大降低碰撞率。

  2. 信任度可以精确反映车辆是否遵守指令,是有效的不确定性度量。

  3. 结果表明信任模型对实现系统安全性和效率都是必要的。

综上,文章通过详细案例研究证明了在智能交通系统中采用可信任的协同控制方法的必要性和有效性。
