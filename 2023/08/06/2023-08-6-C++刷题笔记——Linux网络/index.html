C/C++C++Linux网络多线程

C++刷题笔记

宁康2023-08-062023-08-07

Part1

  • 多进程能够支持并发服务器的原因是 父子进程共享文件描述符表

  • **多线程实现高并发服务器主控线程要传递给子线程的核心数据是 ** 用于与客户端通信的文件描述符

    • 因为线程之间栈区是独立的,子线程内部并没有用于与客户端通信的文件描述符,需要主控线程传递给子进程

  • 编写UDP服务器需要的步骤 ：创建套接字（socket），绑定ip和端口（bind），接收数据与发送数据（recvfrom，sendto)

    • UDP不需要建立连接,所以不需要listen和accept处理

    • 面试常考题目：TCP/IP和UDP协议的区别？

      • 答案：TCP和UDP的区别 (zhihu.com))

      • 总的来说，TCP/IP 提供了一种可靠的、有序的、以连接为基础的数据传输方式，而 UDP 提供了一种无连接的、不可靠的、可能无序的数据传输方式。TCP/IP 更适合对数据完整性有高要求的应用，例如 Web 浏览器和电子邮件客户端，而 UDP 更适合对速度有高要求的应用，例如在线视频和游戏。

  • 判断：无论是Windows系统还是Linux系统，都可以通过ifconfig命令检测网络连接状态

    • 错误，Linux系统中可以用ifconfig 命令来查看和配置网络接口，而windows系统使用ipconfig查看和配置网络接口。而检测网络连接状态命令是ping.

Part2

  • 浏览器和服务器间通信必须遵循http协议,但是传输层要选择使用TCP协议
    • 由于使用bs架构,那么协议选择不灵活,应用层固定使用http协议,传输层使用TCP,浏览器会自动与服务器建立TCP连接
  • 信息系统安全的含义有
    • 机密性
    • 完整性
    • 可用性
    • 可追溯性
  • Linux常用的版本控制系统有
    • git
      • Git：Git 是一个分布式的版本控制系统，由 Linux 的创造者 Linus Torvalds 开发。Git 允许每个开发者拥有完整的项目历史记录，这让开发者可以离线工作，也使得合并和冲突解决变得更容易。Git 还支持非线性开发，允许创建多个独立的分支，并在需要时进行合并。
    • svn
      • SVN (Subversion)：SVN 是一个集中式的版本控制系统，由 Apache Software Foundation 维护。在 SVN 中，所有的更改都提交到一个中心服务器，开发者需要从服务器获取最新的版本。SVN 对文件和目录的重命名和移动有良好的支持，但它的分支和合并功能不如 Git 强大。
    • CVs
      • **CVS (Concurrent Versions System)**：CVS 也是一个集中式的版本控制系统，是现代版本控制系统的先驱之一。然而，CVS 的功能相对较弱，比如它无法原子性地提交跨越多个目录的更改，也不支持文件和目录的重命名。因此，现在大多数项目已经转向了更强大的系统，如 Git 或 SVN。
  • linux知识：NAT模式下，只要物理机能访问网络，虚拟机便能访问网络。在网络地址转换 (NAT, Network Address Translation) 模式下，虚拟机和物理机共享同一个网络连接。虚拟机发送的所有网络流量都会经过物理机，并由物理机转发到网络上。这意味着只要物理机能访问网络，虚拟机也就能访问网络。
